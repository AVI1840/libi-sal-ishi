# LIBI — סקריפט הדגמה
## למצגות בפני גורמי מקצוע ומקבלי החלטות

> **זמן כולל:** 15–20 דקות  
> **קהל יעד:** מנהלי רשויות, אנשי משרד הרווחה, קופות חולים, ביטוח לאומי  
> **מה צריך:** מחשב נייד, חיבור אינטרנט, Docker פועל

---

## פתיחה (2 דקות)

**מה לומר:**

> "בישראל יש כ-1.2 מיליון אזרחים ותיקים. רובם זכאים לשירותים — פיזיותרפיה, פעילות חברתית, ציוד מסייע. אבל רובם לא מממשים את הזכויות שלהם. לא כי הם לא רוצים — כי המערכת מורכבת מדי.
>
> LIBI היא שכבת AI שיושבת מעל המערכות הקיימות ועושה בדיוק דבר אחד: מחברת בין האזרח לשירות שמגיע לו, בלי בירוקרטיה."

---

## תרחיש 1 — קליטת אזרח חדש (5 דקות)

**פתח:** `http://localhost:3001` (Case Manager Dashboard)

**מה לעשות:**
1. הצג את לוח הבקרה הראשי — 75 לקוחות, 3 התראות דחופות
2. לחץ על "לקוח חדש" → הצג את אשף הקליטה
3. מלא את השאלות (ניידות, חושים, תחומי עניין, חלום מרכזי)
4. לחץ "סיים" → הצג את הפרופיל שנוצר

**מה להדגיש:**
- "הפרופיל הזה נוצר תוך 5 דקות. בלי טפסים, בלי מסמכים."
- "המערכת זיהתה אוטומטית: רמת סיעוד 3, 96 יחידות, 5 שירותים מומלצים"
- "הפרסונה — 'שואפת לשמור על עצמאות' — תנחה את כל ההמלצות הבאות"

**נקודת עצירה לשאלות:** "מה קורה היום בתהליך הקליטה שלכם?"

---

## תרחיש 2 — שיחה עם AI (5 דקות)

**פתח:** `http://localhost:3000` (Web Client)

**מה לעשות:**
1. הצג את הממשק — פשוט, גדול, בעברית
2. לחץ על "שיחה עם לימור" (AI Companion)
3. הקלד: **"הגב כואב לי כבר כמה ימים"**
4. הצג את תגובת ה-AI — זיהוי, הצעה, הזמנה
5. הקלד: **"כן, בבקשה"**
6. הצג: הזמנה בוצעה, תזכורת נשלחה, משפחה עודכנה

**מה להדגיש:**
- "שרה לא הורידה אפליקציה. לא מילאה טופס. דיברה בעברית פשוטה."
- "ה-AI בדק את הארנק, מצא ספק מתאים, קבע תור — הכל תוך 90 שניות"
- "הבן קיבל עדכון אוטומטי. המשפחה בלולאה, בלי שיחות טלפון"

**הדגמה נוספת (אופציונלי):**  
הקלד: **"אני מרגיש לבד"** — הצג זיהוי בדידות + הצעת פעילות חברתית

---

## תרחיש 3 — לוח בקרה CRM (3 דקות)

**חזור ל:** `http://localhost:3001`

**מה לעשות:**
1. הצג את ה-KPI dashboard: 62% שירותי מניעה, 78% ניצול ארנק
2. לחץ על "פעולות דחופות" — הצג 3 התראות
3. לחץ על "שרה כהן — ציון בדידות נמוך"
4. הצג: פרופיל + המלצת פעולה + כפתור "בצע"
5. לחץ "בצע" — הצג אישור

**מה להדגיש:**
- "מתאמת אחת מנהלת 75 לקוחות. היום היא מנהלת 30."
- "המערכת לא מחליפה את המתאמת — היא מכפילה את היכולת שלה"
- "כל פעולה מתועדת. יש audit log מלא לביקורת"

---

## תרחיש 4 — פורטל ספקים (2 דקות, אופציונלי)

**פתח:** `http://localhost:3002` (Vendor Portal)

**מה לעשות:**
1. הצג dashboard ספק: 156 הזמנות, דירוג 4.9
2. הצג ניהול שירותים — הוספה, עריכה, זמינות
3. הצג תשלומים — סטטוס, היסטוריה

**מה להדגיש:**
- "הספק מקבל הזמנות אוטומטיות. אין שיחות טלפון."
- "הרשות רואה את כל הספקים, הדירוגים, ואיכות השירות"

## סיכום ושאלות (3 דקות)

**מה לומר:**

> "מה שראיתם היום הוא דמו אסטרטגי — לא מוצר מוגמר. זה הוכחת היתכנות לכך שאפשר לבנות מערכת הפעלה לזכויות שעובדת.
>
> השאלה שאנחנו שואלים: האם הבעיה הזו שווה לפתור? ואם כן — מה הצעד הבא?"

**שאלות מנחות לדיון:**
- "כמה זמן לוקח היום לקלוט לקוח חדש אצלכם?"
- "כמה אחוז מהזכויות מנוצלות בפועל?"
- "מה הכי מסורבל בתהליך הנוכחי?"

## טיפים להדגמה

**לפני:**
- ודא ש-Docker פועל: `docker compose ps`
- בדוק health: `curl http://localhost:8000/health`
- פתח את שלושת הממשקים מראש בטאבים נפרדים

**במהלך:**
- דבר לאט — הקהל לא טכני
- הדגש את הערך העסקי, לא הטכנולוגיה
- אם משהו לא עובד — "זה דמו, בואו נמשיך לחלק הבא"

**אחרי:**
- שלח קישור לריפו
- הצע פגישת עומק טכנית נפרדת
- בקש פידבק ספציפי: "מה הכי רלוונטי לכם?"

## API Demo (לקהל טכני)

```bash
# בדיקת בריאות
curl http://localhost:8001/health

# רשימת תרחישי דמו
curl http://localhost:8001/api/v1/demo/scenarios

# הרצת תרחיש — בדיקת בוקר
curl -X POST http://localhost:8001/api/v1/demo/scenarios/morning_checkin/run

# KPI Dashboard
curl http://localhost:8002/api/v1/lev/crm/dashboard

# המלצות אישיות
curl http://localhost:8002/api/v1/lev/recommendations/demo-user-001

# חישוב סבסוד
curl -X POST http://localhost:8002/api/v1/lev/subsidy/calculate \
  -H "Content-Type: application/json" \
  -d '{"service_base_price": 200, "content_world": "health_function"}'
```
