# LIBI — ארכיטקטורה

## שירותים

| שירות | Port | תפקיד |
|-------|------|--------|
| Integration Gateway | 8000 | API Gateway, auth, routing |
| AI Agent | 8001 | שיחה, בריאות, זיכרון, התראות |
| Marketplace | 8002 | ארנק, שירותים, הזמנות, ספקים |
| PostgreSQL | 5432 | מסד נתונים ראשי (3 schemas) |
| Redis | 6379 | sessions, cache |

## Schemas במסד הנתונים

```
libi_db
├── shared      — users, family_members, audit_log
├── ai_agent    — conversations, messages, health_readings, alerts, semantic_memories
└── marketplace — wallets, vendors, services, bookings, transactions
```

## תפקידי משתמש (RBAC)

| תפקיד | תיאור |
|-------|--------|
| `senior` | האזרח הוותיק |
| `family` | בן משפחה |
| `case_manager` | מתאמת שירות |
| `vendor` | ספק שירות |
| `authority_manager` | מנהל רשות |
| `admin` | מנהל מערכת |

## עולמות תוכן (Content Worlds)

מבוסס על החלטת ממשלה להזדקנות מיטבית:

| עולם | סבסוד ברירת מחדל |
|------|-----------------|
| שייכות ומשמעות | 100% |
| בריאות ותפקוד | 100% |
| חוסן ועצמאות | 50% |
| מוצרים וטכנולוגיה מסייעת | 50% |
| שירותי בית | 20% |

## מנוע סבסוד דינמי

```
מחיר בסיס
    × (1 - אחוז סבסוד לפי עולם תוכן)
    × בונוסים: פעילות קבוצתית, תוסף הכנסה, ציון בדידות
= מחיר סופי ללקוח
```

## זרימת נתונים — הזמנת שירות

```
1. אזרח מדבר עם AI
2. AI מזהה צורך → בודק ארנק → מחפש שירות
3. AI מציע → אזרח מאשר
4. AI קורא ל-Marketplace API → יוצר הזמנה
5. יחידות מוקפאות בארנק
6. ספק מקבל הודעה
7. לאחר ביצוע → יחידות מנוכות, ספק מקבל תשלום
8. מתאמת רואה הכל ב-dashboard
```
